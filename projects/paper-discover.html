Paper Discover

Explore papers by discipline, publication date, and AI-powered insights across topics and abstracts.

View live app →

What Paper Discover Does

Paper Discover is a lightweight research exploration tool that helps users understand what’s being published, where, and how fields are evolving — without needing to manually sift through thousands of papers.

Instead of keyword-only search, the app lets users:

  • Browse papers by discipline
  • See publication trends over time
  • Get high-level summaries and insights across abstracts and topics

The goal is not to replace deep reading, but to make the discovery phase faster and more intuitive.

How the Product Works (Conceptually)

At a high level, Paper Discover has three main layers:

  • A simple, responsive interface for exploring papers
  • A backend that normalizes and serves publication data
  • Optional AI-powered insights to summarize trends across many papers

Each part is intentionally kept modular so the system is easy to extend and reason about.

Data Ingestion & Curation

Paper data is sourced from the Springer Open Access API and periodically synced into the app’s database.

Rather than querying external APIs directly from the browser, all data is:

  • Fetched server-side
  • Cleaned and normalized
  • Stored in a structured database

This allows the app to:

  • Respond quickly to user queries
  • Support aggregation (e.g. counts by date or discipline)
  • Avoid rate limits and inconsistent upstream responses

An admin-only sync endpoint handles ingestion, making it easy to refresh or expand the dataset without impacting users.

Exploring Papers in the App

Once data is ingested, users can explore it in a few ways:

Browsing by Discipline

Each paper can belong to multiple disciplines. These are stored as structured metadata rather than free text, which makes it possible to:

  • Filter reliably
  • Rank popular disciplines
  • Compare fields side by side

Viewing Publication Trends

To help users see how research evolves over time, the app aggregates papers by publication date and surfaces this data visually. This makes it easy to answer questions like:

  • “Is this field growing quickly?”
  • “When did interest peak?”

Reading Abstracts in Context

Instead of isolating each paper, the interface encourages scanning across abstracts to get a sense of common themes before diving deeper.

AI-Powered Insights (Optional by Design)

Paper Discover includes an optional AI insights feature that summarizes patterns across multiple abstracts.

This is intentionally:

  • Not a replacement for reading
  • Not positioned as authoritative
  • Clearly labeled as a synthesis tool

The goal is to help users orient themselves quickly — especially when entering an unfamiliar field.

I designed this feature conservatively, focusing on:

  • Aggregation rather than interpretation
  • Transparency over confidence
  • Human judgment remaining central

Why I Designed It This Way

A few guiding principles shaped the product:

  • Clarity over cleverness — The system favors explicit data flows and readable interfaces over hidden magic.
  • Server-side control of data — All reads and writes go through the backend, which keeps security, validation, and evolution manageable.
  • Extensibility — New data sources, filters, or insight layers can be added without rewriting the core system.
  • Respect for the research process — The tool accelerates discovery but doesn’t pretend to replace expertise.

Technical Notes (For Those Who Want Them)

For readers interested in implementation details:

  • The frontend is built with React and Next.js
  • Data is stored in PostgreSQL (via Supabase)
  • All data access goes through API routes, rather than directly from the browser
  • Aggregations and summaries are handled server-side for consistency and performance

These choices reflect a preference for maintainable, production-oriented systems over quick demos.

What I’d Explore Next

If this were taken further, I’d be interested in:

  • Supporting multiple publication sources
  • Adding saved views for specific research interests
  • Personalized recommendations for authenticated users based on their research interests
  • Surfacing changes over time within individual disciplines
  • Improving evaluation and transparency around AI-generated insights
